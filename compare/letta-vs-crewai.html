Letta vs CrewAI

Letta and CrewAI represent two different bets on what agents need most. Letta (formerly MemGPT) bets on memory — its tiered memory architecture keeps agents stateful across hours, days, weeks. CrewAI bets on simplicity — role-based agents that you can ship in an afternoon, with memory as a side concern. Both are open source, both work with any LLM.

The verdict

Use Letta if your agent's value comes from accumulating state — customer assistants that know a user's history, research agents that build knowledge over time, personal AI that remembers preferences across weeks. Use CrewAI if you want fast, stateless multi-agent pipelines — content generation, automated research reports, batch processing where each run is independent. Many teams use both: CrewAI for stateless batch jobs, Letta for the always-on customer-facing assistant.

How we tested

We built two systems with each framework. (1) A "weekly research report" agent that runs once and outputs a report — CrewAI won (4 lines of code vs 80, finished in 2 min vs 8 min). (2) A "personal AI assistant" that needs to remember user preferences across weeks — Letta won (CrewAI has no built-in memory, required manual DB integration). The frameworks are complementary, not competitive.
