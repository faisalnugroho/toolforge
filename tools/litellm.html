Stop rewriting your API calls for every model

LiteLLM is a proxy and SDK that translates one OpenAI-style call into 100+ providers (Anthropic, Gemini, Bedrock, Azure, Groq, Ollama). Swap models with a string change, load-balance across keys, and track spend per project.

Who it's for: Platform and backend engineers who want provider portability, failover, and cost control without vendor lock-in.

Key features

Unify 100+ models

One interface for OpenAI, Anthropic, Gemini, Bedrock, and more.

Balance Fallbacks

Route to a backup model automatically when one provider is down or rate-limited.

Track Spend logs

Per-key, per-project token and cost accounting out of the box.

Proxy OpenAI-compatible

Drop-in server your existing OpenAI SDK can hit.

The honest take

✓ What works

  • Eliminates provider lock-in
  • Trivial model swaps and A/B tests
  • Excellent cost observability
  • Open source and lightweight

✗ What doesn't

  • You still need to manage keys/quotas
  • Some provider quirks leak through
  • Proxy adds a tiny latency hop
  • Advanced routing needs config effort

Verdict

LiteLLM is the cheapest insurance against LLM vendor surprises. If you call more than one provider, run it yesterday.
