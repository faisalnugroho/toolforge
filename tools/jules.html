An agent that does the tedious coding for you

Jules is Google's asynchronous coding agent, built on Gemini 2.5. You drop an issue or a TODO, and Jules spins up a cloud VM, explores your codebase, writes a plan, implements the change, runs tests, and opens a pull request for your review. It is not an autocomplete — it is a junior engineer that works in the background.

Who it's for: Busy engineering teams drowning in bug triage, dependency upgrades, and small feature requests. Best for GitHub repos where you'd otherwise context-switch away from deep work.

Key features

Asynchronous agent

Assign a task and Jules works off-cycle in a hosted environment — you get a PR, not a chat bubble.

Codebase planning

Before touching code, Jules summarizes its plan and reasoning so you can course-correct early.

Multi-repo edits

Edits span many files and services, then opens a pull request with a clear diff and test run.

Gemini 2.5 brain

Backed by Google's frontier model for strong reasoning on large, unfamiliar codebases.

The honest take

✓ What works

  • Genuinely asynchronous — frees you from context-switching
  • Reads the whole repo before editing, fewer blind changes
  • Clean PRs with test runs and explanations
  • Free during preview

✗ What doesn't

  • Preview waitlist still gates access
  • Slower than interactive pair programming for tiny fixes
  • Google-ecosystem leanings in tooling
  • Occasional over-engineering on vague prompts

Verdict

If your team sits on a backlog of 'should fix eventually' tickets, Jules is the closest thing to hiring a night-shift engineer in 2026. Pair it with Cursor or Cline for interactive work and you cover both modes.
