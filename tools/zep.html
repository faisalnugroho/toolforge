Give your LLM app long-term memory

Zep is a purpose-built memory service for LLM apps and agents. Instead of stuffing the whole conversation into the prompt, you store facts in a graph and retrieve only what's relevant — faster, cheaper, and more accurate. It's the 'memory' behind production chatbots.

Who it's for: Developers shipping chatbots, agents, and copilots that need to recall user context.

Key features

Graph memory Facts, not blobs

Facts are extracted and stored as a knowledge graph for precise retrieval.

Auto extraction Zero-shot

Zep's LLM extracts entities and relationships from conversation automatically.

Fast retrieval Millisecond recall

Optimized stores return relevant memory in milliseconds, not seconds.

Open source Self-hostable

Deploy on your own infra; your data never leaves your account.

The honest take

✓ What works

  • Makes agent memory actually work in production
  • Graph approach beats naive RAG for facts
  • Open source + self-hostable
  • Solid docs and SDKs
  • Free tier for prototyping

✗ What doesn't

  • Adds an infra dependency to manage
  • Best value shows at scale, not toy apps
  • Smaller community than LangChain
  • Needs LLM calls for extraction (cost)

Verdict

If you're building an agent that needs to 'remember' users across sessions, Zep is one of the most production-ready memory layers available. Start free, self-host, and scale as your user base grows.
